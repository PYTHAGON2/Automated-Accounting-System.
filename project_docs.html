Introduction

Automated Accounting System

AutoSys is a modern, cross-platform accounting solution built with Flutter and Firebase. It provides real-time transaction tracking, user authentication, and administrative oversight within a sleek, performant interface.

State Management

Powered by Provider for reactive UI updates and centralized business logic.

Cloud Native

Real-time data synchronization across devices using Firebase Cloud Firestore.

Secure Access

Role-based authentication system supporting both standard users and admins.

Project Structure

The codebase follows a clean-architecture inspired Flutter layout, separating models, state, and UI.

Directory Tree
lib/
├── main.dart           // Application entry point & routing
├── app_state.dart      // Core business logic & State Management

Key Files & Responsibilities

State

app_state.dart

The "brain" of the app. Handles Firebase Auth listeners, Firestore stream subscriptions, and global state calculations (e.g., total balance).

UI

main.dart

Initializes Firebase and wraps the entire app in a ChangeNotifierProvider. Manages the high-level routing flow based on auth status.

Data

models.dart

Defines User, Admin, and Transaction classes. Includes fromFirestore and toFirestore serialization logic.

Styling

theme.dart

Centralizes the design system, defining the color palette, button styles, and typography for a consistent look and feel.

Core Logic Highlights

Real-time Data Fetching

The AppState class uses a StreamSubscription to listen to Firestore changes. When a user adds a transaction, the UI updates instantly everywhere without a page refresh.

app_state.dart (Simplified Snapshots)
void _listenToTransactions() {
  _transactionSubscription = _firestore
    .collection('transactions')
    .where('user', isEqualTo: _currentUser?.username)
    .snapshots()
    .listen((snapshot) {
      _transactions = snapshot.docs
        .map((doc) => Transaction.fromFirestore(doc.id, doc.data()))
        .toList();
      notifyListeners();
    });
}

Role-Based Routing

The application determines the home screen dynamically based on the currentRole defined in the state.

main.dart (getHome Logic)
Widget _getHome(AppState state) {
  if (state.currentRole == 'user') return const UserDashboard();
  if (state.currentRole == 'admin') return const AdminDashboard();
  return const LoginScreen();
}
